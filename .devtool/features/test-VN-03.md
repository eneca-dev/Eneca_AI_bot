# test-VN-03 Подключение ассистента (Recall) + дедуп

Тестируем по правилам «Человек + Нейросеть». На каждый подтверждённый дефект —
отдельный тикет `bug-VN-NN` со ссылкой сюда.

## Цель
Отправка ссылки → ассистент идёт на встречу; повторная отправка той же встречи →
«уже на встрече» (один бот на встречу); приём ассистента из «комнаты ожидания»;
освобождение блокировки после завершения.

## Контекст реализации (для проверяющего ИИ)
- `recall_client.join_meeting`: `_normalize_meeting_url` → ключ; `_active_meetings` (lock, TTL 3 ч).
- Дубликат → `MeetingAlreadyJoinedError` (без HTTP в Recall) → бот отвечает «уже на встрече».
- Блокировка снимается из webhook на `bot.done` / fatal (`release_meeting_lock`).

## Среда
Дедуп/нормализация/освобождение блокировки — проверены как **чистая логика**
(`scratchpad/probe_dedup.py`, 8/8, дубликат ловится ДО HTTP → живой бот не создаётся).
Реальное подключение (S1/S2/S3) — нужен настоящий Teams-созвон + живой Recall-бот:
server.py на :8000 + ngrok + Azure endpoint. ИИ читает лог (`join_meeting`, `Skip join`,
`Released meeting lock`).
