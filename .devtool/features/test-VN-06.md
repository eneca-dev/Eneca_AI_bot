# test-VN-06 Артефакты DOCX + доставка в Teams

Тестируем по правилам «Человек + Нейросеть». На каждый подтверждённый дефект —
отдельный тикет `bug-VN-NN` со ссылкой сюда.

## Цель
Рендер протокола и таблицы транскрипта в DOCX, загрузка в Supabase Storage
(signed URL), доставка сообщения в Teams со ссылками на файлы.

## Контекст реализации (для проверяющего ИИ)
- `render_report_docx` / `render_transcript_docx` → байты; `storage_client.upload_meeting_artifact` (бакет, signed URL, `rstrip("?")`).
- `_format_artifact_links` → markdown-ссылки; `teams_sender.send_message` (текст + блок ссылок).
- Регрессы: битые ссылки с хвостовым `?`; signed-URL вместо публичного.

## Среда
Артефактный уровень — локально (`scratchpad/probe_vn06.py`): `render_report_docx`/
`render_transcript_docx` из реальной строки `meeting_reports` → `storage_client.upload_meeting_artifact`
→ проверка signed-URL (HTTP GET, формат, TTL). Доставка в Teams (S1/S4/S6) — реальный чат.
